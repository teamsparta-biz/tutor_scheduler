# 역할 기반 접근제어

> 관리자(admin)와 강사(instructor) 역할을 구분하여 접근을 제어한다

## 목표

- 사용자별 역할이 DB에 저장됨
- 프론트엔드에서 역할에 따라 메뉴/화면이 다르게 표시됨
- 백엔드 API에서 역할에 따라 접근이 제한됨

## 상세

- [ ] users 테이블에 role 필드 활용 (admin / instructor)
- [ ] 백엔드: 요청 시 JWT에서 사용자 정보 추출 → 역할 확인 미들웨어
- [ ] 백엔드: 관리자 전용 API에 역할 검증 적용 (강사 CRUD, 교육 CRUD, 배정 등)
- [ ] 프론트엔드: 역할에 따라 사이드바 메뉴 분기
- [ ] 프론트엔드: 관리자 전용 페이지 접근 시 강사는 리디렉트
- [ ] 첫 번째 가입 사용자를 admin으로 설정하는 방법 결정

## 참고

- Supabase RLS 정책으로 DB 레벨에서도 보호 가능

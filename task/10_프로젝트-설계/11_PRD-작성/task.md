# PRD 작성

> 프로젝트 요구사항 문서(PRD)를 작성하여 MVP 범위를 명확히 정의한다

## 목표

- 제품 비전, 대상 사용자, 핵심 기능, Non-goals가 문서화됨
- 사용자 플로우(화면 흐름)가 정의됨
- 성공 기준이 명시됨

## 상세

- [ ] 제품 비전 한 문단 작성
- [ ] 대상 사용자 역할 정의 (운영팀 관리자, 강사)
- [ ] 핵심 기능 목록 상세화 (강사 관리, 교육 일정, 배정, 캘린더)
- [ ] Non-goals 목록 작성 (MVP에서 제외할 것)
- [ ] 사용자 플로우 정의 (관리자 플로우, 강사 플로우)
- [ ] 업무 규칙 정리 (하루 1개 교육, 반별 배정 등)
- [ ] 성공 기준 정의

## 참고

- 인터뷰 결과: 프로젝트 README.md 개요 섹션 참조
- 결과물 위치: `output/PRD.md`
